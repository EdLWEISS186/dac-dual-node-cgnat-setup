## SBT / ERC-5192 Behavior

The final contract is designed as a locked wallet badge:

```text
locked(tokenId) = true
transferFrom() reverts
safeTransferFrom() reverts
approve() reverts
setApprovalForAll() reverts
```

A wallet can mint only one badge. If the wallet profile changes, the same badge is updated through `updateBadge()`.

## Wallet Status Badge Flow

```text
User checks wallet
↓
Wallet Intelligence Layer generates profile
↓
Badge preview appears
↓
Web checks Wallet Status contract
↓
If no badge exists:
  Mint Badge · 1 DACC
↓
If badge exists and contract profile changed:
  Update Badge · 0.001 DACC
↓
If badge exists and is current:
  Badge Up To Date
```
